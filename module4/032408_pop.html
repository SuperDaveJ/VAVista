Lock Command Intent button

The intent of the LOCK command is to avoid having two or more routines alter the same data node(s) simultaneously. However, the LOCK only prevents two jobs from LOCKing the same node at the same time. When a LOCK is executed, M checks the LOCK table to see if the node is already LOCKed by another job. If it is not, an entry is added to the LOCK table with the job number of the routine.
