Lock Command Use button

Using the LOCK command only prevents another user from LOCKing the same node; a locked node can be changed by a routine that does not LOCK that node. Therefore, the LOCK command is only an effective “safety” feature if all programmers LOCK nodes before they create or edit them. All forms of the LOCK command (except the incremental and decremental LOCKs) first unlock nodes that have been locked by the current job or partition.
