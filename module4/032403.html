2.4.2    CLOSE Command

  • Direct use of the CLOSE command is prohibited. Use the supported ^%ZIS* namespaced routines. (Exemptions:  Kernel, Mailman and VA FileMan)
  • Closes the device or file designated by its argument. This command will free the device or file that was previously allocated by the OPEN command. This command is vendor-specific in its argument values, meaning it can affect portability.
  • CLOSE may only be used directly by FileMan , MailMan, and Kernel or through Kernel-supplied utilities. This is so VA code is not implementation-specific
